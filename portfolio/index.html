i. abstract

I am a mathematics student at New York University building software for robotics, numerical methods, and Unix-based engineering workflows. This page is structurfed like a short technical document so I can show the work clearly and attach figures, screenshots, and diagrams as the projects are documented.

1. projects

1.1. Autonomy and Localization, NYU Robotics Design Team, NASA Lunabotics

Stack. ROS 2, Python, C++, OpenCV, Linux, Docker

This is the strongest engineering work I have right now. The project lives inside a team rover stack and includes autonomy, localization, perception, launchability, Linux systems work, and the integration work that makes the whole thing usable instead of just impressive on paper.

  • Worked in a multi-package ROS 2 workspace spanning autonomy, localization, command handling, and interfaces.
  • Contributed to camera and AprilTag-related localization workflows.
  • Built a headless Ubuntu 24.04 image for the Jetson Orin Nano and tuned the footprint to cut bloat, latency, and swap overhead.
  • Integrated a Docker-based ROS 2 wrapper so rover-side software was more controlled and reproducible.
  • Helped tighten rover-to-ground-control traffic and packet flow.
  • I also help with the marketing and presentation side of RDT so the work can actually be shown and explained.

1.2. Ground Control, Manual Command Bridging, and Gazebo Simulation

Stack. ROS 2 + Gazebo, Python, Linux, Docker

This is the side of the work that makes the rover easier to test and easier to drive. It is less glamorous than autonomy, but it is closer to what real systems work feels like: command paths, simulation setup, launch files, and debugging how pieces talk to each other.

1.3. Newton and gradient methods on the Rosenbrock surface

Stack. MATLAB

In numerical analysis work, I implemented experiments around Newton's method, gradient-based updates, and visualization for nonlinear surfaces in two variables. This is the kind of math-to-code work I want in the portfolio because it shows gradients, Hessians, convergence behavior, failure cases, and visual explanation in the same place.

f(x,y) ≈ 10e4 f(x,y) ≈ 10e2 f(x,y) ≈ 10e-1

This uses the same MATLAB turbo color map as the exported surface figures. Purple is lower and orange-red is higher.

1.4. Bronco Space / Project BILL-EE

Stack. ROS 2, Python, C++, Linux, Docker

This project let me work closer to the controls and robotics-mechanics side of things. It is another example of how I like work where math actually has to become behavior in a physical system.

  • Implemented inverse-kinematic systems for a robotic arm and used them to improve efficiency.
  • Worked with proportional-integral-derivative control ideas inside a ROS 2 workflow.

1.5. FIRST Robotics Competition, Team 589

Stack. Java, WPILib, Arduino, REV Robotics, National Instruments, LimeLight, Photonvision

This is where a lot of my software identity got built. I was leading students, writing code, fixing problems at competitions, and learning how to make autonomous and teleoperated systems work under pressure.

  • Led the software subsystem for a team of 15 students and taught runtime concepts, troubleshooting, and error handling.
  • Wrote more than 12,000 lines of Java across two years for autonomous and teleoperated robot behavior.
  • Integrated Arduino, REV Robotics, and National Instruments hardware with WPILib.
  • Improved autonomous performance through real-time localization.
